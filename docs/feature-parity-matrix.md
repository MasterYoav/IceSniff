# Feature Parity Matrix

This matrix tracks whether a capability exists in the shared engine and how each interface exposes it.

| Capability | Shared service | CLI | Desktop | Notes |
| --- | --- | --- | --- | --- |
| Interactive shell workflow | Partial | Partial | Planned | Implemented as `shell [capture-file]` plus default no-arg startup into a persistent CLI session with `open`, `status`, `mode`, live-capture commands, and analysis commands against the current capture |
| Inspect capture file metadata | Yes | Yes | Planned | Implemented as `inspect <capture-file>` |
| Open capture file | Partial | Partial | Planned | Shared file loading exists for metadata and PCAP/PCAPNG packet records |
| List packets | Yes | Yes | Planned | Implemented as `list <capture-file> [limit] [--filter <expr>]` for PCAP and common PCAPNG enhanced-packet blocks with derived source/destination/protocol/info columns |
| Inspect packet detail | Partial | Partial | Planned | Implemented as `show-packet <capture-file> <packet-index>` for PCAP and common PCAPNG enhanced-packet blocks with minimal Ethernet/ARP/IPv4/TCP/UDP/ICMP decoding, a byte-range-aware field tree, and DNS/HTTP/TLS application metadata |
| Capture stats | Partial | Partial | Planned | Implemented as `stats <capture-file> [--filter <expr>]` for PCAP and common PCAPNG enhanced-packet blocks with summary counts by link/network/transport |
| Basic filtering | Yes | Yes | Planned | Shared semantics currently support packet keys like `protocol=...`, `service=...`, `port=...`, `ip=...`, `endpoint=...`, `host=...`, plus broader field-aware predicates such as `http.reason~=...`, `dns.answer_count=...`, and `tls.handshake_length>=...`, combined with boolean composition, comparison operators, case-insensitive text matching, and backward-compatible comma-AND syntax |
| Conversations | Partial | Partial | Planned | Implemented as `conversations <capture-file> [--filter <expr>]` with bidirectional endpoint normalization, service guessing, directional packet counts, and request/response totals for recognized protocols |
| Streams and transactions | Partial | Partial | Planned | Implemented as `streams <capture-file> [--filter <expr>] [--stream-filter <expr>]` with client/server roles, TCP session-state counts, matched transaction counters, ordered session-event timelines, HTTP pipelining notes, repeated TLS handshake cycle counts/state, TLS alert summaries, out-of-order/retransmission/gap notes, and row-level stream filtering including derived session booleans like pipelining, timeline presence, and reassembly-issue detection |
| Transaction enumeration | Partial | Partial | Planned | Implemented as `transactions <capture-file> [--filter <expr>] [--transaction-filter <expr>]` with ordered HTTP request/response rows and TLS handshake rows derived from reassembled TCP streams, including structured HTTP transfer/body metadata, TLS ALPN/alert/certificate progression metadata, and row-level transaction filtering with derived HTTP/TLS aliases and completeness/alert predicates |
| Stable JSON schema versioning | Yes | Yes | Planned | CLI `--json` report outputs now include top-level `schema_version` (`v1`) for automation compatibility |
| Stable CLI error codes | Yes | Yes | Planned | CLI failures now emit stable stderr prefixes (`[ISCLI_USAGE]`, `[ISCLI_RUNTIME]`) with consistent exit statuses (`2` usage, `1` runtime) for scripting |
| Live capture | Partial | Partial | Planned | Implemented in the CLI shell as `capture interfaces`, `capture start [interface]`, `capture stop`, and `capture status` through shared `app-services` orchestration backed by `capture-engine` session logic with `tcpdump`-style and `dumpcap` backends, with a live `Id | Time | Source | Destination | Protocol | Info` packet table and a temporary PCAP that becomes the current capture when stopped |
| Save/export capture | Partial | Partial | Planned | Implemented as `save <source-capture-file> <output-capture-file> [--filter <expr>] [--stream-filter <expr>]` plus shell `save <output-capture-file> ...`, writing filtered packet selections or stream-filter-selected packets to a new PCAP through shared services; stream/packet export formats beyond PCAP remain pending |
